Polygon based meshes are very common in computer simulation. You see them all the time in computer simulations and computer games. A common issue in computer simulation is performance. I particular not computing more than we have to during any computer simulation. Although meshes are very complex and useful data structures wouldn't it be nice if when a very detailed mesh is off in the background of some simulation. Methods for assisting with this issue such as this have been around for a couple of years and they are sometimes referred to as Level Of Detail (LOD) algorithms.

Mesh simplification is a LOD method. The goal of the simplification is to remove elements, usually edges or vertices, from a mesh while introducing the least amount of error (non-smoothness). Error can be calculated in many ways, some of the best ways being global volume calculations, however we want something that is very fast and practice. For this work I used difference between the sum of the angles around a vertex and 360 degrees (2pi).

  1. I started with adding a few debugging annotations
    1. Highlighting vertices that are about to be removed
  2. The first step is to construct a priority queue of some sort so that the vertex with the least amount of error can be easily found and removed at each step.
  3. When a vertex is removed the hole in the mesh needs to be re-triangulated. There are a couple of methods to do this. I used the fan algorithm
  4. Vertices can be removed until some specified amount of tolerance. For example don't remove vertices that will cause error > e.
